1. Read: PACKAGE_SUMMARY.txt (2 min)

2. Extract: pm-tool-complete.zip

   $ cd pm-tool-complete

3. Install dependencies:
   $ python -m venv venv
   $ source venv/bin/activate  # or venv\Scripts\activate on Windows
   $ pip install -r requirements.txt

4. Start PM Tool (Terminal 1):
   $ python src/pm_tool_backend.py

5. Start MCP Server (Terminal 2):
   $ python src/mcp_task_agent.py

6. Open browser:
   http://localhost:3000

7. Create a task in the browser

8. Open VS Code and use Copilot:
   Ctrl+Shift+I → "Get my current task"

❓ COMMON QUESTIONS
══════════════════════════════════════════════════════════════════════

Q: Do I need to modify any files?
A: No! Everything works as-is.

Q: What if port 3000 is already in use?
A: Edit src/pm_tool_backend.py and change port=3000 to another port.

Q: Can I use a different database?
A: Yes, modify pm_tool_backend.py to connect to your database.

Q: Will my data persist?
A: Currently no, data resets on restart. Add a database for persistence.

Q: Can I integrate with my real PM tool?
A: Yes! Update PM_TOOL_API in .env to point to your tool's API.

Q: Does it work on Windows?
A: Yes! Use venv\Scripts\activate instead of source venv/bin/activate

Q: Is it production-ready?
A: For local use yes. Add authentication for production.

See PACKAGE_SUMMARY.txt for more FAQs!
